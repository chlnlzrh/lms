```
1. Test Suite Setup (Organizing tests for execution)
2. Baseline Establishment (Current state, performance baseline)
3. Test Execution (Running tests, capturing results)
4. Result Analysis (Pass/fail analysis, issue identification)
5. Issue Escalation (Defect logging and escalation)
6. Reporting (Status and metrics reporting)
```

```
6. Closure Validation (Confirming resolution)
```


## ESSENTIAL PROMPTS FOR QA

### Test Case Generation
- "Generate comprehensive test cases for [feature]. Include happy path, error cases, and edge cases."
- "Create test cases for [user workflow]. Cover all decision paths and data variations."
- "Generate boundary value test cases for [input field] with these constraints."

### Defect Analysis
- "Analyze this defect and identify likely root causes. [defect description]"
- "This defect is occurring in [scenario]. What could be causing it? Suggest testing."
- "We're seeing this pattern in defects: [pattern]. What systemic issue might this indicate?"

### Test Automation
- "Write a test automation script for [scenario] using [technology]. Include assertions and error handling."
- "Debug this failing test. [test code and error]. What's going wrong?"
- "Refactor this test suite for better maintainability and reusability."

### Performance Testing
- "Analyze these performance test results. [data]. Identify bottlenecks and recommend optimization."
- "Create a performance test plan for [application]. What metrics should we track?"

### Test Strategy
- "Create a comprehensive test strategy for [project]. Cover scope, approach, resources."
- "We need to reduce test cycle time by 50%. Create an optimization strategy."
- "Develop a regression test strategy for [system]. What tests should be automated?"
